# review-gate — レビューの評価基準と手順（評価基準の"実装接続"はここ）

## 破綻ゲート（GATE CHECK）— これが落ちたら他の評価以前に修正必須
- **G1 因果・論理・時間整合**（重み最大）
- **G2 話者識別の明瞭さ**
- **G3 文法・統語の破綻なし**
- いずれか「1以下」なら総合点に関わらず大幅改稿扱い。

## 25項目ルーブリック（100点満点・重み付き / know_how_explore 由来）
- ミクロ層: 文体一貫性(4) / 語彙(3) / 文法(4) / 句読点リズム(4) / 描写の情報制御(4) / 比喩(2) / 台詞の自然さ(5) / 話者識別(3)
- マクロ層: 視点焦点化(4) / プロット骨格(6) / 因果時間整合(7) / ペーシング(6) / テーマ対立軸(5) / キャラアーク(6) / 伏線設置回収(6)
- 総合層: 没入移入(5) / サスペンス(4) / ジャンル適合(4)
- 閾値: **80+ 公開候補** / 60-79 改稿ループ継続 / 59- 設計から作り直し

## 追加criterion（evaluation-revolution-lab の発見済み候補・shadow運用）
- knowledge-state-fairness（読者とキャラの知識遷移が跨話で追跡可能か）← ontology_check と連動
- causal-chain-visibility（因果の連鎖が追える）
- theme-turn-embodiment（主題転換が summary でなく scene pressure で見えるか）
- image-before-verdict（概念判断より先に身体像/衝突像が立つか）
- ending-afterimage-strength（余韻で閉じているか）
- dialogue-purpose-density（台詞が情報/関係変化/欲望衝突のいずれかを担うか）

## method（どう読むか）
- **axis-separated-review**: 読者体験レビュー ⊥ 技術整合性レビュー を分離。
- **multi-pass-draft-review**: Pass1 因果認知 / Pass2 Writing Pack適合 / Pass3 読者シミュ。
- **persona 2ラウンド制**: 1周目=表層欠陥、2周目=1周目修正で露出した次層。3周目以降はROI逓減で原則投入しない。

## レビューは感想で終わらせない（採否まで出す）
各指摘に対し必ず: 採用 / 条件付 / 却下 / learning送り の判定 ＋ 反映先（draft/scene/packet/arc/bible/promises のどれに遡上するか）を記す。
